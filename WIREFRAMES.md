# FloatNote 와이어프레임

## 1. 목적

이 문서는 FloatNote의 MVP 화면 구조를 고정하기 위한 와이어프레임 문서다.  
목표는 예쁜 시안을 만드는 것이 아니라, 사용자가 어떤 순서로 화면을 보고 무엇을 입력하며 언제 이동하는지를 명확히 하는 것이다.

## 2. 화면 세트

MVP 기준 화면은 3개면 충분하다.

1. 메인 플로팅 노트 창
2. 첫 실행 안내
3. 설정 화면

목록 화면, 대시보드, 사이드바는 넣지 않는다.  
FloatNote의 핵심은 "호출 즉시 입력"이므로 메인 화면은 단일 창 구조로 유지한다.

## 3. 공통 규칙

- 창은 작은 유틸리티 앱처럼 보여야 한다.
- 상단은 얇고, 본문이 화면의 대부분을 차지해야 한다.
- `Created`, `Updated`는 항상 하단 고정이다.
- 창이 뜨면 즉시 본문에 포커스가 있어야 한다.
- 좌우 이동은 화면 전체가 바뀌는 느낌이 아니라, 같은 창 안에서 노트가 넘어가는 느낌이어야 한다.
- 첫 노트의 왼쪽은 no-op
- 마지막 노트의 오른쪽은 새 노트 생성

## 4. 화면 1: 메인 플로팅 노트 창

### 4-1. 기존 노트가 열렸을 때

```text
┌──────────────────────────────────────────────────────────────┐
│ FloatNote                         12 / 37            [⚙] [×] │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  오늘 확인할 것                                              │
│                                                              │
│  - Tauri global shortcut 충돌 확인                           │
│  - auto-save debounce 500ms                                  │
│  - settings는 별도 진입보다 가볍게                          │
│                                                              │
│  ## next                                                     │
│  이동 단축키 후보 정리                                       │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│ Created 2026-03-13 10:14    Updated 2026-03-13 10:42        │
└──────────────────────────────────────────────────────────────┘
```

### 의도

- 상단 좌측은 제품 이름만 둔다.
- 상단 중앙/우측에는 현재 노트 위치를 둔다.
  - 예: `12 / 37`
  - 사용자는 목록 없이도 현재 위치를 감각적으로 이해할 수 있다.
- 우상단에는 최소한의 액션만 둔다.
  - `설정`
  - `닫기`
- 본문은 가장 넓고 방해 없는 입력 영역이다.
- 하단 메타바는 항상 고정이다.

### 왜 이런 구조인가

- 사용자는 먼저 본문을 봐야 한다.
- 노트 위치는 중요하지만 본문보다 앞서면 안 된다.
- 메타데이터는 필요하지만 조용해야 한다.

## 5. 화면 1-A: 빈 새 노트

마지막 노트에서 오른쪽으로 넘어가면 새 노트가 열린다.

```text
┌──────────────────────────────────────────────────────────────┐
│ FloatNote                         38 / 38            [⚙] [×] │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  제목 없이 바로 입력 시작                                    │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│ Created just now          Updated just now                   │
└──────────────────────────────────────────────────────────────┘
```

### 의도

- placeholder는 설명보다 행동을 유도해야 한다.
- "새 노트를 만들었습니다" 같은 모달은 띄우지 않는다.
- 사용자는 빈 본문을 보면 이미 새 노트임을 이해한다.

## 6. 화면 1-B: 첫 노트의 왼쪽 경계

첫 노트에서 이전 이동을 누르면 화면은 그대로 유지된다.

```text
┌──────────────────────────────────────────────────────────────┐
│ FloatNote                          1 / 37            [⚙] [×] │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  첫 번째 노트 내용                                           │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│ Created 2026-03-01 09:10    Updated 2026-03-01 09:12        │
└──────────────────────────────────────────────────────────────┘
          ↑
     짧은 흔들림 또는 얇은 경계 하이라이트
```

### 의도

- 경계에서는 모달, 토스트, 경고 문구를 쓰지 않는다.
- 너무 많은 피드백은 흐름을 끊는다.
- 짧은 흔들림이나 얇은 플래시 정도면 충분하다.

## 7. 메인 창의 세부 구조

### 상단 바

```text
[앱 이름]             [현재 위치]             [설정] [닫기]
```

규칙은 아래와 같다.

- 앱 이름은 브랜드 인지용이다.
- 현재 위치는 생성 순서 기준 인덱스다.
- 설정 버튼은 작고 조용해야 한다.
- 닫기 버튼은 마우스 사용자용 보조 수단이다.
- 키보드 사용자의 기본 종료는 `Escape`다.

### 본문 영역

```text
[텍스트 입력 영역]
```

규칙은 아래와 같다.

- 여기가 화면의 80% 이상을 차지해야 한다.
- 툴바 버튼을 붙이지 않는다.
- 제목 입력칸을 따로 두지 않는다.
- 처음부터 본문 한 칸으로 시작한다.
- 나중에 인라인 렌더링이 붙더라도 같은 영역 안에서 해결해야 한다.

### 하단 메타바

```text
[Created ...]                              [Updated ...]
```

규칙은 아래와 같다.

- 항상 보인다.
- 본문보다 시선 우선순위가 낮아야 한다.
- 폰트 크기와 대비를 한 단계 낮춘다.

## 8. 메인 창의 인터랙션 흐름

### 열기

1. 전역 단축키 입력
2. 창이 현재 작업 위에 뜬다
3. 마지막으로 본 노트가 열린다
4. 커서는 본문에 있다

### 이전 이동

1. 이전 노트 이동 단축키 입력
2. 더 오래된 노트로 전환
3. 현재 위치 숫자 갱신
4. 자동 저장 유지

### 다음 이동

1. 다음 노트 이동 단축키 입력
2. 더 새로운 노트가 있으면 그 노트로 전환
3. 마지막 노트라면 새 노트 생성
4. 현재 위치 숫자 갱신

### 닫기

1. `Escape` 또는 전역 단축키 재입력
2. 현재 상태 저장
3. 창 닫힘
4. 원래 앱으로 포커스 복귀

## 9. 화면 2: 첫 실행 안내

첫 실행 안내는 별도 풀스크린이 아니라, 메인 창 위에 가볍게 얹히는 구조가 좋다.

```text
┌──────────────────────────────────────────────────────────────┐
│ FloatNote                          1 / 1             [⚙] [×] │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│         ┌──────────────────────────────────────────┐         │
│         │ FloatNote 시작하기                       │         │
│         │                                          │         │
│         │ 1. Control + A 로 열기                   │         │
│         │ 2. 바로 입력                             │         │
│         │ 3. 좌우 이동 키로 노트 넘기기            │         │
│         │                                          │         │
│         │ 기본 단축키는 설정에서 바꿀 수 있음      │         │
│         │                                          │         │
│         │             [시작하기]                   │         │
│         └──────────────────────────────────────────┘         │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│ Created just now          Updated just now                   │
└──────────────────────────────────────────────────────────────┘
```

### 의도

- 첫 실행에서도 메인 창 구조를 그대로 보여준다.
- 사용자는 "이게 어떤 앱인지"보다 "어떻게 바로 쓰는지"만 알면 된다.
- 설명은 세 줄이면 충분하다.

### 피해야 할 것

- 여러 장짜리 온보딩
- 삽화 중심 소개
- 기능 소개용 캐러셀

## 10. 화면 3: 설정 화면

설정은 무거운 Preferences 창보다 가벼운 시트 또는 작은 별도 창이 낫다.

```text
┌──────────────────────────────────────────────┐
│ Settings                             [닫기] │
├──────────────────────────────────────────────┤
│                                              │
│ Global Shortcut                              │
│ [ Control + A               ] [변경]         │
│ 기본 호출 단축키를 바꿀 수 있습니다.         │
│                                              │
│ Note Navigation                              │
│ [ Prev Shortcut              ] [변경]        │
│ [ Next Shortcut              ] [변경]        │
│ 아직 기본값 확정 전                          │
│                                              │
│ Window Size                                  │
│ [ Medium                     v ]             │
│                                              │
│ 충돌 감지                                    │
│ 현재 단축키가 시스템과 충돌하면              │
│ 대체 조합을 제안합니다.                      │
│                                              │
└──────────────────────────────────────────────┘
```

### 의도

- 설정은 자주 머무는 장소가 아니다.
- 가장 중요한 항목은 전역 단축키다.
- 이동 단축키도 이 화면에서 같이 관리한다.
- 창 크기 설정은 MVP에서 허용 가능한 최소 옵션이다.

## 11. 상태별 와이어프레임 우선순위

### 가장 먼저 그려야 하는 상태

1. 기존 노트가 열린 메인 창
2. 새 노트가 열린 메인 창
3. 첫 실행 안내

### 나중으로 밀어도 되는 상태

1. 인라인 렌더링 적용 상태
2. 단축키 충돌 상세 안내
3. 세밀한 애니메이션 시안

## 12. 레이아웃 비율 제안

메인 창 기준 권장 비율은 아래 정도가 적당하다.

- 상단 바: 전체 높이의 8~10%
- 본문: 전체 높이의 78~82%
- 하단 메타바: 전체 높이의 10~12%

권장 폭/높이는 아래처럼 시작하면 된다.

- 기본 폭: 640~760px
- 기본 높이: 460~560px

이유는 아래와 같다.

- 너무 작으면 입력 집중이 깨진다.
- 너무 크면 "잠깐 꺼내 쓰는 툴"이 아니라 메인 앱처럼 보인다.

## 13. 와이어프레임 결정 사항

- 메인 경험은 단일 플로팅 창이다.
- 별도 목록 화면은 없다.
- 상단은 얇게, 본문은 넓게, 하단 메타바는 고정한다.
- 새 노트 생성은 오른쪽 이동의 결과다.
- 첫 노트 왼쪽 이동은 no-op다.
- 첫 실행 안내는 메인 창 위 오버레이다.
- 설정은 작고 단순한 창이다.
- 인라인 마크다운 렌더링은 2단계에서 같은 본문 영역 위에 붙인다.

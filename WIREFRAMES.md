# FloatNote 와이어프레임

## 1. 목적

이 문서는 FloatNote의 메인 화면을 `content-first` 방향으로 다시 고정하기 위한 와이어프레임 문서다.  
목표는 기능을 많이 보여주는 것이 아니라, 본문이 가장 먼저 읽히는 플로팅 문서 경험을 만드는 것이다.

## 2. 화면 세트

MVP 기준 화면은 아래 3개면 충분하다.

1. 메인 플로팅 노트 창
2. 첫 실행 안내 callout
3. 설정 popover

목록 화면, 대시보드, 사이드바는 넣지 않는다.

## 3. 공통 규칙

- 상단 크롬은 얇고 조용해야 한다.
- 본문을 별도 카드 안에 다시 넣지 않는다.
- 설명 라벨은 메인 화면에서 최대한 제거한다.
- 상단 액션은 최소 개수만 유지한다.
- `Created`, `Updated`는 항상 하단 고정이다.
- 창이 뜨면 즉시 본문에 포커스가 있어야 한다.
- 첫 노트의 왼쪽은 no-op다.
- 마지막 노트의 오른쪽은 새 노트 생성이다.

## 4. 화면 1. 메인 플로팅 노트 창

### 4-1. 기존 노트

```text
┌──────────────────────────────────────────────────────────────┐
│ FloatNote                               ◀  3 / 3  ▶   ⚙  ＋ │
│                                                              │
│  작업 위에 잠깐 뜨는 문서                                     │
│                                                              │
│  - 본문이 먼저 보여야 한다                                    │
│  - 설명 라벨은 메인 화면에서 지운다                           │
│  - 상단 액션은 적고 가벼워야 한다                             │
│                                                              │
│  ## next                                                     │
│  와이어프레임과 시안을 새 방향으로 다시 정리                  │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│ Created 2026-03-13 06:43          Updated 2026-03-13 12:53  │
└──────────────────────────────────────────────────────────────┘
```

### 의도

- 상단은 `브랜드 + 위치 + 최소 액션`만 남긴다.
- 본문이 화면 대부분을 차지한다.
- `LAST VIEWED NOTE`, `LIVE MARKDOWN STYLING` 같은 보조 문구는 보이지 않는다.
- 저장 배지, 큰 버튼, 과한 설명은 메인 화면에서 제거한다.

│                                                              │
│  제목 없이 바로 입력 시작                                    │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│ Created just now                    Updated just now         │

### 의도

- 새 노트 생성은 별도 모달 없이 빈 캔버스 자체로 이해되어야 한다.
- placeholder는 설명보다 행동을 유도해야 한다.

## 6. 화면 1-B. 첫 노트의 왼쪽 경계

```text

│ FloatNote                               ◀  1 / 4  ▶   ⚙  ＋ │
│▌                                                             │
│  첫 번째 노트 내용                                           │
│                                                              │
│  더 왼쪽으로는 가지 않는다                                    │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│                                                              │
│ Created 2026-03-01 09:10          Updated 2026-03-01 09:12  │
└──────────────────────────────────────────────────────────────┘
```

### 의도

- 경계에서는 모달, 토스트, 경고 문구를 쓰지 않는다.
- 아주 짧은 좌측 pulse 또는 얇은 하이라이트면 충분하다.

## 7. 메인 창 구조

### 상단 바

```text
[브랜드]                         [이전] [현재 위치] [다음] [설정] [새 노트]
```

규칙은 아래와 같다.

- 본문보다 더 무거우면 안 된다.
- 텍스트 버튼이 너무 크거나 둥근 칩처럼 반복되면 안 된다.
- `닫기`는 macOS 기본 창 동작과 `Escape`로 충분하다.
- `저장`은 상시 큰 버튼이나 배지로 둘 필요가 없다.

### 본문 영역

```text
[문서형 입력 표면]
```

규칙은 아래와 같다.

- 화면의 대부분을 차지한다.
- 별도 입력 카드로 한 번 더 감싸지 않는다.
- 제목 필드를 따로 만들지 않는다.
- 입력과 렌더링은 같은 표면 안에서 해결한다.

### 하단 메타바

```text
[Created ...]                              [Updated ...]
```

규칙은 아래와 같다.

- 항상 보인다.
- 대비와 크기는 본문보다 한 단계 낮춘다.
- 메타데이터가 제품 UI처럼 튀어 보이면 안 된다.

## 8. 메인 창 인터랙션 흐름

### 열기

1. 전역 단축키 입력
2. 창이 현재 작업 위에 뜬다
3. 마지막으로 본 노트가 열린다
4. 커서는 본문에 있다

### 이전 이동

1. 이전 이동 단축키 입력
2. 더 오래된 노트로 전환
3. 현재 위치 갱신

### 다음 이동

1. 다음 이동 단축키 입력
2. 더 새로운 노트가 있으면 전환
3. 마지막 노트라면 새 노트 생성

### 닫기

1. `Escape` 또는 전역 단축키 재입력
2. 현재 상태 저장
3. 창 닫힘
4. 원래 앱으로 포커스 복귀

## 9. 화면 2. 첫 실행 안내

첫 실행 안내는 화면을 점령하는 온보딩보다 작은 callout가 적절하다.

```text
┌──────────────────────────────────────────────────────────────┐
│ FloatNote                               ◀  1 / 1  ▶   ⚙  ＋ │
│                                                              │
│                                            ┌──────────────┐  │
│                                            │ ⌃A 로 열기   │  │
│                                            │ 바로 입력    │  │
│                                            │ 앞뒤로 이동  │  │
│                                            │      [확인]  │  │
│                                            └──────────────┘  │
│                                                              │
│  제목 없이 바로 입력 시작                                    │
│                                                              │
│                                                              │
│                                                              │
│ Created just now                    Updated just now         │
└──────────────────────────────────────────────────────────────┘
```

### 의도

- 첫 실행에서도 본문 구조를 가리지 않는다.
- 사용자가 알아야 하는 핵심만 짧게 보여준다.

## 10. 화면 3. 설정 popover

설정은 작은 popover나 얕은 시트가 적절하다.

```text
                            ┌──────────────────────────────┐
                            │ Shortcuts                    │
                            │ Toggle     Control + A       │
                            │ Older      Control + Shift+← │
                            │ Newer      Control + Shift+→ │
                            │ Window     Medium            │
                            │                    [Done]    │
                            └──────────────────────────────┘
```

### 의도

- 메인 흐름을 완전히 끊지 않는다.
- 가장 중요한 항목은 전역 단축키다.
- 설정 화면 자체가 또 다른 앱처럼 느껴지면 안 된다.

## 11. 상태별 우선순위

### 가장 먼저 구현해야 하는 상태

1. 기존 노트가 열린 메인 창
2. 새 노트가 열린 메인 창
3. 첫 실행 callout

### 나중으로 밀어도 되는 상태

1. 세밀한 애니메이션
2. 충돌 감지 상세 안내
3. 고급 설정 분기

## 12. 레이아웃 비율 제안

- 상단 크롬: 전체 높이의 6~8%
- 본문: 전체 높이의 82~86%
- 하단 메타바: 전체 높이의 8~10%

권장 시작 크기는 아래와 같다.

- 기본 폭: 700~860px
- 기본 높이: 500~620px

## 13. 결정 사항

- 메인 경험은 단일 플로팅 창이다.
- 별도 목록 화면은 없다.
- 메인 화면은 `기능 설명 UI`가 아니라 `조용한 문서`처럼 보여야 한다.
- 상단은 최소화하고 본문이 가장 넓어야 한다.
- 새 노트 생성은 오른쪽 이동의 결과다.

# FloatNote 기획 문서

## 1. 제품 한 줄 정의

`FloatNote`는 전역 단축키로 즉시 띄우는 플로팅 메모 앱이다.  
마크다운 문법으로 빠르게 기록하고, 키보드만으로 이전 노트와 새 노트 사이를 오가며, 최종적으로는 노션처럼 입력 내용이 바로 서식으로 반영되는 경험을 목표로 한다.

## 2. 문제 정의

- 작업 중 메모를 남기려면 앱 전환 비용이 크다.
- IDE, 브라우저, 터미널 위에서 바로 떠야 흐름이 끊기지 않는다.
- 짧은 메모도 서식이 필요할 때가 많아 마크다운 렌더링이 필요하다.

## 3. 목표

- 기본 전역 단축키 `Control + A`로 어디서든 메모 창을 열고 닫을 수 있다.
- 전역 단축키는 사용자가 설정에서 언제든 변경할 수 있다.
- 메모 창은 다른 앱 위에 플로팅된다.
- 본문은 마크다운 문법으로 작성된다.
- 최종 목표는 노션처럼 입력 중 서식이 바로 반영되는 인라인 렌더링이다.
- 하단에는 `Created`, `Updated` 시간이 항상 표시된다.
- 창이 열린 상태에서 이전/다음 노트 이동 단축키로 노트를 넘길 수 있다.
- 노트는 생성 순서 기준으로 왼쪽이 오래된 노트, 오른쪽이 더 새로운 노트다.
- 마지막 노트에서 오른쪽으로 이동하면 새 노트를 만든다.

## 4. 핵심 사용자 시나리오

### 시나리오 A: 빠른 기록

1. 사용자가 코딩 중 `Control + A`를 누른다.
2. 현재 보고 있던 앱 위에 FloatNote가 뜬다.
3. 마지막으로 보던 노트가 열리거나, 노트가 없으면 빈 노트가 열린다.
4. 메모를 입력하고 `Escape` 또는 다시 `Control + A`로 창을 닫는다.

### 시나리오 B: 노트 탐색

1. FloatNote가 떠 있는 상태에서 이전 노트 이동 단축키를 누른다.
2. 더 오래된 노트가 열린다.
3. 다음 노트 이동 단축키를 누르면 더 새로운 노트로 이동한다.
4. 마지막 노트에서 오른쪽으로 이동하면 새 빈 노트가 생성되어 열린다.

### 시나리오 C: 서식 반영

1. 사용자는 마크다운 문법으로 입력한다.
2. 최종 목표는 문법이 별도 미리보기 없이 본문에서 바로 서식으로 반영되는 것이다.
3. 구현 복잡도가 높으면 MVP에서는 원문 입력을 우선 제공하고, 인라인 렌더링은 다음 단계로 넘긴다.
4. 하단 고정 영역에 생성일과 수정일이 보인다.

## 5. 기능 요구사항

### 필수 기능(MVP)

- 전역 단축키로 열기/닫기
  - 기본값: `Control + A`
  - 사용자가 설정에서 반드시 변경 가능해야 한다.
  - OS 또는 다른 앱과 충돌하면 대체 단축키를 제안해야 한다.
- 플로팅 창
  - 항상 위(`always on top`)
  - 포커스를 받되, 빠르게 닫을 수 있어야 함
- 노트 생성/조회/수정
- 마지막으로 본 노트 다시 열기
- 마크다운 문법 입력 지원
- 이전 노트 / 다음 노트 이동
  - 왼쪽은 더 오래된 노트
  - 오른쪽은 더 새로운 노트
- 첫 노트에서 왼쪽 이동 시 현재 노트 유지
- 마지막 노트에서 새 노트 생성
- `Created`, `Updated` 타임스탬프 하단 고정 표시
- 자동 저장
  - 입력 중 debounce 저장
  - 노트 이동 시 즉시 저장

### 단계별 도입

- 1단계 MVP
  - 플로팅 창
  - 마지막으로 본 노트 진입
  - 마크다운 원문 입력
  - 생성 순서 기반 좌우 이동
  - 자동 저장
  - 하단 메타데이터 표시
- 2단계
  - 노션 스타일 인라인 마크다운 렌더링

### 제외 기능(초기 버전)

- 클라우드 동기화
- 태그/폴더
- 협업
- 이미지 업로드
- 첨부파일

## 6. UX 초안

### 창 레이아웃

- 상단: 앱 이름 또는 현재 노트 상태
- 본문: 마크다운 입력 영역
- 하단: `Created: YYYY-MM-DD HH:mm` / `Updated: YYYY-MM-DD HH:mm`

### 편집 방식 제안

- 기본은 편집 가능한 본문 영역을 사용한다.
- 최종 목표는 노션처럼 본문 안에서 서식이 바로 반영되는 인라인 렌더링이다.
- 다만 MVP 우선순위는 "입력 안정성"이므로, 초기 버전은 마크다운 원문 입력만 먼저 제공할 수 있다.
- 별도 분할 미리보기 화면은 기본 방향으로 채택하지 않는다.

### 키보드 동작

- `Control + A`: 기본 열기/닫기 단축키
- 이전 노트 이동 단축키: 기본값 추후 확정
- 다음 노트 이동 단축키: 기본값 추후 확정, 마지막이면 새 노트 생성
- `Escape`: 창 닫기
- `Command/Control + S`: 수동 저장

## 7. 데이터 모델 초안

```ts
type Note = {
  id: string;
  body: string;          // markdown source
  createdAt: string;     // ISO timestamp
  updatedAt: string;     // ISO timestamp
};
```

### 정렬 규칙

- 기본 정렬은 생성 순서 기준
- 앱을 열면 마지막으로 본 노트를 연다.
- 좌/우 이동은 현재 노트의 생성 순서를 기준으로 탐색
- 왼쪽은 더 오래된 노트, 오른쪽은 더 새로운 노트
- 첫 노트의 왼쪽은 no-op
- 마지막 노트의 오른쪽은 "새 노트 생성"

## 8. 기술 방향 제안

### 1차 권장 스택

- `Tauri`
- `React`
- `TypeScript`
- 1단계 입력 UI: 일반 텍스트 입력 기반 에디터
- 2단계 인라인 렌더링: `Tiptap` / `ProseMirror` 계열 검토
- 저장소: 로컬 파일 또는 SQLite

### 이유

- 플로팅 데스크톱 앱에 적합하다.
- Electron보다 가볍게 시작할 수 있다.
- 로컬 우선 메모 앱과 잘 맞는다.

## 9. 저장 방식 제안

### 옵션 A: SQLite

- 장점
  - 정렬, 검색, 업데이트 시간이 명확하다.
  - 노트 수가 늘어도 관리가 쉽다.
- 단점
  - 파일 직접 편집은 어렵다.

### 옵션 B: Markdown 파일 저장

- 장점
  - 데이터가 사람이 읽기 쉽다.
  - 백업과 버전 관리가 쉽다.
- 단점
  - 정렬/인덱싱 로직을 직접 다뤄야 한다.

### 초기 선택

- MVP는 `SQLite`를 우선 권장
- 외부 export는 나중에 `.md` 파일로 지원

## 10. 리스크

- `Control + A`는 일부 환경에서 이미 강하게 점유된 단축키일 수 있다.
- 전역 단축키 등록은 macOS 권한 및 접근성 설정 이슈가 있을 수 있다.
- 플로팅 창 포커스 처리에 따라 입력 경험이 불안정해질 수 있다.
- 인라인 마크다운 렌더링은 일반 텍스트 입력보다 구현 난도가 높다.

## 11. MVP 완료 기준

- 전역 단축키로 창이 뜬다.
- 앱을 열면 마지막으로 본 노트가 열린다.
- 노트를 작성하면 자동 저장된다.
- 마크다운 원문 입력이 가능하다.
- 하단에 `Created`, `Updated`가 보인다.
- 이전/다음 이동 단축키로 생성 순서 기준 이동이 가능하다.
- 마지막 노트 오른쪽 이동에서 새 노트가 생성된다.

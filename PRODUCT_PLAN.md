# FloatNote 기획 문서

## 1. 제품 한 줄 정의

`FloatNote`는 전역 단축키로 즉시 띄우는 플로팅 메모 앱이다.  
마크다운으로 작성하고, 렌더링된 결과를 바로 확인하며, 키보드만으로 이전 노트와 새 노트 사이를 빠르게 이동한다.

## 2. 문제 정의

- 작업 중 메모를 남기려면 앱 전환 비용이 크다.
- IDE, 브라우저, 터미널 위에서 바로 떠야 흐름이 끊기지 않는다.
- 짧은 메모도 서식이 필요할 때가 많아 마크다운 렌더링이 필요하다.

## 3. 목표

- `Control + A`로 어디서든 메모 창을 열고 닫을 수 있다.
- 메모 창은 다른 앱 위에 플로팅된다.
- 본문은 마크다운으로 저장되고 렌더링된다.
- 하단에는 `Created`, `Updated` 시간이 항상 표시된다.
- 창이 열린 상태에서 `Control + Left` / `Control + Right`로 노트를 넘길 수 있다.
- 마지막 노트에서 `Control + Right`를 누르면 새 노트를 만든다.

## 4. 핵심 사용자 시나리오

### 시나리오 A: 빠른 기록

1. 사용자가 코딩 중 `Control + A`를 누른다.
2. 현재 보고 있던 앱 위에 FloatNote가 뜬다.
3. 마지막으로 보던 노트가 열리거나, 노트가 없으면 빈 노트가 열린다.
4. 메모를 입력하고 `Escape` 또는 다시 `Control + A`로 창을 닫는다.

### 시나리오 B: 노트 탐색

1. FloatNote가 떠 있는 상태에서 `Control + Left`를 누른다.
2. 이전 노트가 열린다.
3. `Control + Right`를 누르면 다음 노트로 이동한다.
4. 마지막 노트에서 `Control + Right`를 누르면 새 빈 노트가 생성되어 열린다.

### 시나리오 C: 읽기 모드

1. 사용자가 작성한 마크다운이 본문 영역에서 렌더링된다.
2. 링크, 리스트, 코드 블록, 체크박스 정도는 MVP에서 지원한다.
3. 하단 고정 영역에 생성일과 수정일이 보인다.

## 5. 기능 요구사항

### 필수 기능(MVP)

- 전역 단축키로 열기/닫기
  - 기본값: `Control + A`
  - 단, OS 또는 다른 앱과 충돌 시 변경 가능해야 한다.
- 플로팅 창
  - 항상 위(`always on top`)
  - 포커스를 받되, 빠르게 닫을 수 있어야 함
- 노트 생성/조회/수정
- 마크다운 저장 및 렌더링
- 이전 노트 / 다음 노트 이동
- 마지막 노트에서 새 노트 생성
- `Created`, `Updated` 타임스탬프 하단 고정 표시
- 자동 저장
  - 입력 중 debounce 저장
  - 노트 이동 시 즉시 저장

### 제외 기능(초기 버전)

- 클라우드 동기화
- 태그/폴더
- 협업
- 이미지 업로드
- 첨부파일

## 6. UX 초안

### 창 레이아웃

- 상단: 앱 이름 또는 현재 노트 상태
- 본문: 마크다운 편집/렌더링 영역
- 하단: `Created: YYYY-MM-DD HH:mm` / `Updated: YYYY-MM-DD HH:mm`

### 편집 방식 제안

- 기본은 편집 가능한 본문 영역을 사용한다.
- 렌더링은 실시간 미리보기 또는 보기 모드 전환 중 하나로 구현한다.
- MVP 우선순위는 "입력 안정성"이므로, 초안 기준으로는 다음 방식이 적합하다.
  - 좌측: 마크다운 원문 편집
  - 우측: 렌더링 결과
- 창이 작은 경우에는 편집/미리보기 토글로 자동 전환한다.

### 키보드 동작

- `Control + A`: 창 열기/닫기
- `Control + Left`: 이전 노트
- `Control + Right`: 다음 노트, 마지막이면 새 노트 생성
- `Escape`: 창 닫기
- `Command/Control + S`: 수동 저장

## 7. 데이터 모델 초안

```ts
type Note = {
  id: string;
  body: string;          // markdown source
  createdAt: string;     // ISO timestamp
  updatedAt: string;     // ISO timestamp
};
```

### 정렬 규칙

- 기본 정렬은 생성 순서 기준
- 좌/우 이동은 현재 노트의 순서를 기준으로 탐색
- 마지막 노트의 오른쪽은 "새 노트 생성"

## 8. 기술 방향 제안

### 1차 권장 스택

- `Tauri`
- `React`
- `TypeScript`
- 마크다운 렌더링: `react-markdown` 또는 동급 라이브러리
- 저장소: 로컬 파일 또는 SQLite

### 이유

- 플로팅 데스크톱 앱에 적합하다.
- Electron보다 가볍게 시작할 수 있다.
- 로컬 우선 메모 앱과 잘 맞는다.

## 9. 저장 방식 제안

### 옵션 A: SQLite

- 장점
  - 정렬, 검색, 업데이트 시간이 명확하다.
  - 노트 수가 늘어도 관리가 쉽다.
- 단점
  - 파일 직접 편집은 어렵다.

### 옵션 B: Markdown 파일 저장

- 장점
  - 데이터가 사람이 읽기 쉽다.
  - 백업과 버전 관리가 쉽다.
- 단점
  - 정렬/인덱싱 로직을 직접 다뤄야 한다.

### 초기 선택

- MVP는 `SQLite`를 우선 권장
- 외부 export는 나중에 `.md` 파일로 지원

## 10. 리스크

- `Control + A`는 일부 환경에서 이미 강하게 점유된 단축키일 수 있다.
- 전역 단축키 등록은 macOS 권한 및 접근성 설정 이슈가 있을 수 있다.
- 플로팅 창 포커스 처리에 따라 입력 경험이 불안정해질 수 있다.

## 11. MVP 완료 기준

- 전역 단축키로 창이 뜬다.
- 노트를 작성하면 자동 저장된다.
- 마크다운이 정상 렌더링된다.
- 하단에 `Created`, `Updated`가 보인다.
- `Control + Left` / `Control + Right`로 노트를 이동할 수 있다.
- 마지막 노트 오른쪽에서 새 노트가 생성된다.
